User-First Design

User stories:

  1. As a new user, I should be able to register a new account.
    • User visits the form/new page of the micro-blogging application.
    • User fills out their full name, uploads a profile picture, enters a username/handle, and a password.
    • User then clicks the 'Register' button, and their account is now created.
  2. As a normal user, I should be able to follow someone.
    • User visits the message index page, which has a feed of messages from all users.
    • User clicks on the 'Follow' button to the right of the message for whoever they want to follow. The user is now following that person.
    • Your feed now updates with 'You are now following [user]'.
  3. As a normal user, I should be able to post a message.
    • User visits the message new page.
    • User fills out the 'New Message' form with a message that allows for #hashtags and @mentions.
    • User then clicks the 'Post' button that publishes their new message on their account, and this message is now visible to any user that follows them.
  4. As a normal user, I should be able to see messages from other people that I follow.
    • User visits the messages index, which has a feed of messages from users that they follow.
    • User can visit a specific person's page and see messages from just that person. You would need to click on the person's username in the feed.
  5. As a normal user, I should be able to see all messages that use the same hashtag.
    • User visits their account home page with the message feed.
    • User navigates to the top right corner of the page and enters a hashtag to search for.
    • A new page is brought up with a list of messages that contain the hashtag being searched for. If there are no messages with that hashtag, a message appears on the page telling the user there are no messages containing that hashtag.
    • User can also click on a hashtag in the message feed, which brings them to the same page containing a list of messages that used that same hashtag.

Data-First Design

Resources needed for the application and what data each resource would need:

  1. Users
    • Name
    • Username/handle
    • Password
    • Email
    • Profile Picture
    • Messages
  2. Messages
    • Information of user who posted the message (name and username)
    • Date and time posted
    • Contents of the message

Data constraints/issues:

  • There can't be more than one account using the same email address.
  • How will profile picture images be stored in the database?
